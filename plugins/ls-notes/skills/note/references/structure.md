# Vault Structure

The vault uses the PARA method with extensions for periodic notes, knowledge capture, and project management.

- **Projects/**: Active projects with specific outcomes. Each is a folder containing a kanban board file. **REQUIRED:** Use the `project` skill when reading or writing files here.
- **Areas/**: Ongoing areas of responsibility (AI, Business, Development, PKMS, Personal, Work).
- **Resources/**: Reference materials.
  - **Learning Notes/**: Notes from books, podcasts, talks, courses.
  - **Today I Learned/**: Quick technical discoveries.
  - **Articles/**: Saved web articles.
  - **Ideas/**, **Thoughts/**, **Questions/**, **Lists/**, **Principles/**, **Saved Feedback/**.
- **Archive/**: Completed or inactive items.
- **Future/**: Someday/maybe.
- **Systems/**: Life systems and processes.
- **Daily Notes/**: One file per day. **REQUIRED:** Use the `daily-note` skill when reading or writing files here.
- **Templates/**: Content templates.
- **Triage/**: Inbox for uncategorized notes.
